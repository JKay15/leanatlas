## Purpose / Big Picture
Wave-A already enforces a strict execution meta-loop, but five practical gaps remain: deterministic sub-agent invocation routing, discoverable Python/MCP interfaces for Codex use, maintainer-focused upgrade/repair documentation, reviewer-history contradiction controls, and guaranteed doc-policy visibility for provider-invoked agents. This plan makes those constraints machine-checkable in contracts/tests so they are not advisory text only. The end state is that LOOP runs can prove who reviewed what, with which provider/profile, under which instruction scope, and whether reviewer judgments are internally consistent across rounds.

## Glossary
- Agent invocation routing: deterministic resolution from run config to executable command/provider/profile.
- Reviewer-history consistency: cross-iteration check that reviewer claims do not self-contradict without explicit rationale.
- Instruction scope chain: root `AGENTS.md` + nearest scoped `AGENTS.md` / `AGENTS.override.md` references exposed to provider-invoked agents.
- Provider parity: `codex exec` and alternate providers (`--agent-provider`) must satisfy equivalent evidence contracts.

## Decision log
- Keep reviewer-history semantics in Wave execution schema (closest to AI review loop evidence).
- Keep provider neutrality by requiring `agent_provider_id` + resolved command evidence, not codex-only hardcoding.
- Keep human review non-blocking; contradiction detection is audit input and AI-review input, not a hard stop by itself.
